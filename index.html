Multi-modal abnormality detection · CT · MRI · Ultrasound · X-ray

How a scan becomes a finding

An engineering pipeline that takes a DICOM upload, routes it to a modality-specific model, and returns three things: what the abnormality is, where it is, and how confident the model is — with the confidence actually calibrated and the coordinates mapped back to the original scan.

01 · The process

Eight stages, one scan

Every upload flows through these stages. Click any stage to see what happens inside it and which module owns it. The order matters: category selection up front is what makes the rest modality-specific.

1Intake & category ui/RadiologyIntake.jsx · api.py›
The user picks a scan category before uploading. That choice forces routing to the correct specialized model, so detection never depends on missing or mislabeled DICOM tags — the single most reliable way to get modality-specific results.
  • Upload is gated: it unlocks only once a category arms a model.
  • Accepts a single frame, a multi-slice series, or a zipped study.
2Ingestion — DICOM → tensor dicom_ingestion.py›
Reads the DICOM series with pydicom (tolerating corrupted headers and grouping by series), assembles the pixel volume, and normalizes it into a model-ready tensor.
  • HU rescale applied for CT, then window/level clip (verified applied, not assumed).
  • Orientation → RAS and isotropic resampling for volumes; a separate 2D path for US/X-ray.
  • Records both affines (original + preprocessed) so detections can be mapped back later.
  • PHI is de-identified first — identifiers blanked, private tags removed, UIDs remapped (grouping preserved).
3Routing label_taxonomies.py›
The category maps to a registry key (e.g. ct_chest_nodule_seg). Category selection overrides metadata; a metadata-based fallback exists for automated pipelines where no category is supplied.
4Model & inference model_registry.py · inference.py · cxr_rad_dino.py›
The registry builds the right architecture with correct input/output channels (3D U-Net, UNETR, Mask R-CNN, or the RAD-DINO CXR encoder).
  • Sliding-window inference runs volumes patch-by-patch, so full-resolution CT/MR fit in memory.
  • The chest-X-ray path uses microsoft/rad-dino (DINOv2 ViT-B, 518², 37×37 patches).
5Localization confidence_localization.py · explainability.py›
Two regimes, because the datasets supervise localization differently:
  • Mask → boxes (LIDC, BraTS, BUSI): threshold the segmentation, label connected components, emit boxes.
  • Grad-CAM → boxes (RSNA-ICH, CheXpert): the labels are image-level only, so a saliency map is thresholded into boxes. Works for CNNs and ViTs.
6Confidence calibration calibration.py›
Raw softmax/sigmoid outputs are over-confident. Temperature scaling (one scalar fit on validation data) corrects this and is applied at inference. Expected Calibration Error drops ~10–20× on over-confident data; unfitted models default to a no-op.
7Coordinate mapping coordinate_mapping.py›
Boxes come out in the localizer's space (feature grid, or resampled/padded volume). They're mapped back to original scan voxels and patient world millimetres through the MetaTensor affines — which undo orientation flips and resampling in one step.
8Output api.py · /analyze›
A JSON payload with the top finding, per-detection boxes (model / original-voxel / world-mm), the calibrated confidence, the temperature applied, and the intended-use disclaimer. Every inference also appends a tamper-evident audit record (model + code version, a PHI-free content hash of the processed pixels, de-id status).

03 · Architecture

The modules

Ten source modules, each with a single job. Ingestion has no heavy model dependencies; the registry has no ingestion dependencies — the layers stay separable.

Ingestion

dicom_ingestion.py

DICOM → metadata + normalized tensor; series grouping, HU windowing, geometry capture, router.

Taxonomy & routing

label_taxonomies.py

Per-model dataset, task, class map, localization strategy; category→model routing.

Model registry

model_registry.py

Builds each architecture with correct in/out channels; loads weights when available.

Inference

inference.py

Sliding-window volumetric inference with AMP — fits arbitrarily large CT/MR.

Localization

confidence_localization.py

Segmentation logits → connected-component boxes + confidence (temperature-aware).

Explainability

explainability.py

Grad-CAM → boxes for classification models; CNN and ViT (reshape_transform) paths.

Calibration

calibration.py

Temperature scaling: fit, ECE metric, apply, per-model store. Multiclass + multilabel.

Coordinate mapping

coordinate_mapping.py

Detection boxes → original voxel + world mm via affines; undoes resample/orientation/pad.

RAD-DINO CXR

cxr_rad_dino.py

microsoft/rad-dino encoder + multi-label head + ViT Grad-CAM localization.

API

api.py

FastAPI: category-forced routing, calibrated inference, mapped-coordinate output.

Training

training/ (Lightning)

One trainer for all models: DiceCE for segmentation, BCE/CE for classification; fits + stores calibration T; saves loadable weights.

06 · Run it

Setup & commands

Everything runs on CPU. The tests need no data — they generate synthetic DICOM.

# install
pip install -r requirements.txt

# end-to-end plumbing test across all 5 modalities
python scripts/run_smoke_test.py

# focused tests
python scripts/test_coordinate_mapping.py
python scripts/test_sliding_window.py
python scripts/test_calibration.py
python scripts/test_rad_dino.py

# API (category-forced routing + calibrated /analyze)
cd src && CALIBRATION_JSON=calibration.json uvicorn api:app --reload

# UI: ui/RadiologyIntake.jsx (React) — runs standalone with simulated inference,
# or point it at the /analyze endpoint for live results

08 · Intended use & limitations

Read this before trusting an output

Not a medical device

This is research/engineering scaffolding. It is not cleared or validated for diagnosis, triage, or any clinical decision.

Un-trained weights

Models are randomly initialized. Numbers shown anywhere are placeholders until real weights are registered.

Calibration ≠ accuracy

Temperature scaling fixes confidence honesty, not discrimination. It also assumes validation data matches deployment; site/scanner shift needs re-fitting.

Human in the loop

Any real deployment requires trained + externally validated models, regulatory review, PHI handling, and a qualified reader making the call.
